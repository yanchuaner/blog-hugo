## 项目简介

ACM 集训队训练过程中的刷题笔记、算法模板和竞赛复盘仓库。覆盖动态规划、图论、字符串、数据结构等核心竞赛算法。

## 项目背景

加入 ACM 集训队后，需要系统化的刷题记录和可复用的算法模板。这个仓库就是我的"第二大脑"——每次比赛后立即复盘记录。

## 技术架构

```text
算法模板库 (C++ 实现)
    ├── 动态规划
    │   ├── 线性 DP / 区间 DP / 树形 DP
    │   ├── 背包问题 (01/完全/多重)
    │   └── 状态压缩 DP
    ├── 图论
    │   ├── 最短路 (Dijkstra/Bellman-Ford/SPFA/Floyd)
    │   ├── 最小生成树 (Kruskal/Prim)
    │   └── 拓扑排序
    ├── 字符串
    │   ├── KMP
    │   └── Trie 树
    └── 数据结构
        ├── 线段树 / 树状数组
        └── 并查集
```

**技术栈**: C++17 · STL · Makefile · GitHub Actions (自动测试)

## 核心功能

| 功能 | 说明 |
|------|------|
| 算法模板 | 33 类算法，带注释和复杂度分析 |
| 刷题笔记 | 每题记录思路、代码、WA 原因 |
| 竞赛复盘 | Codeforces 比赛后总结与改进点 |
| 自动测试 | GitHub Actions 自动编译运行 |

## 技术难点

### 模板的通用性

一个好的模板既要简洁（比赛时可快速抄写），又要通用（适配不同题目）。需要在"抽象"和"具体"之间找到平衡。

### 刷题效率

Codeforces 从灰名到绿名用了 3 个月，核心是 **质量 > 数量**——每题彻底搞懂比刷 10 题走马观花更有效。
