## 内容规范 v1.0

### 文章 Front Matter 模板

```yaml
---
title: "标题（明确、有吸引力）"
summary: "一句话摘要（用于 RSS 和首页展示，不超过 160 字）"
date: YYYY-MM-DD
lastmod: YYYY-MM-DD
draft: true          # 发布前改为 false
showtoc: true        # 长文章必须开启目录
categories: ["分类"]  # 算法 / 安全 / 开发 / AI
tags: ["标签1", "标签2"]
series: ["系列名"]    # 如果属于系列
comments: true
---
```

### 文章质量标准

1. **正文 ≥ 800 字**（不含代码块）
2. **至少 1 个代码块**（有实际意义的代码，非纯示例）
3. **至少 1 个列表/表格**（增强可读性）
4. **必须有"总结/心得/下一步"段落**
5. **系列文章必须在开头注明"本文是 [系列名] 第 N 篇"并链接回系列索引**

### 系列文章规范

1. 每个系列有 `_index.md` 目录页
2. 系列目录页列出所有文章规划和状态
3. 每篇文章 front matter 必须包含 `series` 字段
4. 系列内文章使用一致的前言格式

### 项目页面规范

1. 位于 `content/projects/项目名/index.md`
2. 必须包含：简介、背景、技术架构、核心功能、技术难点、截图（预留）、总结、GitHub、Roadmap
3. 技术架构用代码块或文本图表示
4. 核心功能用表格呈现

### 知识库笔记规范

1. 位于 `content/notes/分类/`
2. 不同于文章：更结构化、更简短、持续更新
3. 标题即知识点
4. 必须标注来源（如果是学习笔记）
